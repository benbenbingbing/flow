# 流程平台优化计划

## 一、已完成功能 ✅

1. **拖拽式流程设计** - 基于 bpmn-js + Flowable
2. **实体设计** - 实体对应数据库表，含通用字段和业务字段
3. **表单设计** - 实体可设计多个表单，关联到流程节点
4. **流程设计** - 拖拽构建流程，支持节点和流转设置
5. **审批设置** - 支持用户组、角色、个人、表达式、接口动态
6. **多实例配置** - 支持会签（并行）和或签（串行）
7. **待办生成** - 流程触发时自动生成待办
8. **首页展示** - 待办/已办列表

## 二、需要优化的功能 🔧

### 1. 实体数据管理优化
- [x] 新增数据时默认使用第一个流程节点的表单
- [ ] 保存表单时区分"只保存"和"提交并保存"
- [ ] 提交后自动触发流程

### 2. 流程启动优化
- [ ] 提交表单后自动触发流程
- [ ] 未找到审批人时抛异常提示

### 3. 审批规则完善
- [ ] 单签：一人审批通过后，其他待办自动结束
- [ ] 会签：需所有人审批通过才算完成

### 4. 缺失的核心功能
- [ ] 流程撤回（发起人未审批前可撤回）
- [ ] 流程转办（将任务转给他人处理）
- [ ] 流程驳回后重新提交
- [ ] 流程历史查看（审批记录）
