# 领域模型边界

## 目标

动态实体、表单、列表、流程都按三态管理：

- 设计态：编辑中的配置。
- 发布态：可执行的版本快照。
- 运行态：业务数据和流程实例。

运行态只能读取发布态。设计态变更不能影响已发起流程和已保存数据。

## 三态职责

| 状态 | 职责 | 可写入口 | 主要数据 |
| --- | --- | --- | --- |
| 设计态 | 编辑实体、字段、表单、列表、流程、节点配置 | 设计服务 | `entity_definition`、`entity_field`、`entity_form`、`entity_list_config`、`process_definition_config`、`node_config` |
| 发布态 | 记录发布版本，提供运行快照 | 发布服务 | `entity_publish_history`、`process_version_history`、已发布动作、后续表单/列表/关系快照 |
| 运行态 | 写入业务数据，驱动流程实例，执行权限策略 | 运行时服务 | 动态实体表、关系表、流程实例、任务、状态映射 |

## 读写规则

- 设计态可以频繁修改，不直接驱动运行态。
- 发布服务负责生成快照、部署流程、记录版本。
- 发起流程、审批表单、动态数据校验读取发布快照。
- 运行态只保存业务事实，不回写设计态配置。
- 权限策略由策略服务计算，查询层只消费结果。
- 动态 SQL 只能通过运行时入口生成，字段必须来自白名单。

## 当前归属

### 实体

- 设计态：`EntityDefinition`、`EntityField`、`EntityRelation`。
- 发布态：`EntityPublishHistory.fieldsSnapshot`，后续补表单、列表、关系快照。
- 运行态：动态实体表、实体关系运行时、实体状态。

### 表单

- 设计态：`EntityForm`、`EntityFormField`。
- 发布态：节点表单绑定和实体发布快照中的字段协议。
- 运行态：审批详情、数据表单渲染、只读规则。

### 流程

- 设计态：`ProcessDefinitionConfig`、节点配置、草稿动作。
- 发布态：`ProcessVersionHistory`、Flowable 部署、已发布动作。
- 运行态：流程实例、任务、审批记录、进度视图。

### 权限

- 设计态：权限规则配置。
- 发布态：可执行策略。
- 运行态：用户、部门、角色和数据范围计算结果。

## 服务边界

| 模块 | 职责 |
| --- | --- |
| `entity.definition` | 实体设计、字段维护、关系设计 |
| `entity.publish` | 发布快照、版本读取、差异比较 |
| `entity.runtime` | 动态表数据、关系运行时、流程发起 |
| `process.definition` | BPMN、节点配置、发布编排 |
| `process.runtime` | 实例、任务、审批、进度、终止 |
| `permission` | 策略匹配、SQL 条件构建 |

## P2 落地顺序

1. 建实体发布快照读取入口。
2. 发起流程改读实体发布快照。
3. 审批表单改读节点发布快照。
4. 动态数据校验改读实体发布快照。
5. 关系读写统一走运行时入口。
6. 权限策略统一走策略服务。

## 约束

- 新运行时代码不得直接读取草稿配置。
- 新发布代码必须保留版本号和快照内容。
- 新设计态接口不得直接修改运行态数据。
- 兼容旧接口，内部逐步转发到新边界。
