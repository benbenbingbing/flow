## 发现的问题

### 问题 1：角色选择组件渲染为 el-select 而非 el-select-v2（非阻塞）
- **位置**：`workflow-web/src/components/NodeConfigPanel.vue:229-248`
- **现象**：代码写的是 el-select-v2，但运行时 DOM 中角色选择渲染为普通 el-select
- **影响**：功能正常（能选角色、能保存、能回显），但 DOM 与源码不一致
- **建议**：确认 ElementPlus 版本是否支持 el-select-v2；若不支持统一改成 el-select

### 问题 2：表单来源切"实体表单"时多个 select 下拉层同时打开（UX 问题）
- **位置**：`workflow-web/src/components/NodeConfigPanel.vue` 表单 tab
- **现象**：打开"表单来源"下拉后，再打开"选择表单"下拉，之前未关闭的下拉仍停留在页面上，多个 dropdown 重叠
- **影响**：操作干扰，容易点错；功能上保存正确
- **建议**：切下拉前主动关闭其他 el-select-dropdown

### 问题 3：保存后的 BPMN XML 重新导入画布报错（预先存在）
- **位置**：`workflow-web/src/components/VueBpmnDesigner.vue:210`
- **现象**：每次保存/刷新后控制台报错：Failed to execute 'scale' on 'SVGMatrix': The provided float value is non-finite.
- **影响**：画布缩放异常，但节点仍可显示和点击
- **建议**：importXML 成功后先 canvas.resized() 再 zoom，或延迟到容器有尺寸后

### 问题 4：自动跳过 el-switch 点击后 UI 变但数据未同步（较严重）
- **位置**：`workflow-web/src/components/NodeConfigPanel.vue:1356-1363`
- **现象**：点击"是否跳过"开关，UI 显示已开启，但保存后 XML 中 skipNode="false"
- **影响**：用户以为配置了跳过，实际运行时不会跳过
- **建议**：给 el-switch 加 @update:modelValue 或在 @change 中显式同步 advancedForm.skipNode

### 问题 5：保存后 flowable:formData="null" 出现在 XML（数据不洁净）
- **位置**：`workflow-web/src/components/NodeConfigPanel.vue:2829`
- **现象**：实体表单绑定时，updateNodeFormBind 把 flowable:formData 设为 null，导致 XML 出现 flowable:formData="null"
- **影响**：不影响解析，但 XML 不洁净
- **建议**：把 flowable:formData 设为 undefined 而不是 null

### 问题 6：assigneeConfig 默认写入未启用的多实例字段
- **位置**：`workflow-web/src/components/NodeConfigPanel.vue:1632` 和 `@/shared/process-config/index.js:100-123`
- **现象**：未启用多实例时，assigneeConfig 中保存了 collectionSource="interface"、interfaceType="rest" 等默认值
- **影响**：扩展属性冗余，不致命但数据不洁净
- **建议**：buildAssigneeConfig 只保存当前指定方式相关的字段

### 问题 7：流程保存落库后提示消息未更新为"保存成功"（偶发）
- **现象**：点击顶部"保存流程"按钮后，toast 仍显示之前 tab 的"暂存成功"，而非"保存成功"
- **影响**：用户不确定是否已落库
- **建议**：handleSave 中先清空已有 message，再显示"保存成功"；或设置 ElMessage 的实例避免被覆盖

## 新发现的问题（运行时测试）

### 问题 8：实体字段 optionsJson 双重转义（API 调用问题，非 bug）
- **现象**：通过 API 传 optionsJson 字符串时，数据库存了带反斜杠的 `[{\"label\":...}]`
- **根因**：curl/shell 参数转义过度，前端正常保存不会出现此问题
- **结论**：非代码 bug，是 API 测试时参数转义问题，无需修复

### 问题 9：ALL 类型规则命中后停止评估（设计待确认）
- **位置**：`DataPermissionEngine.java:214-219`
- **现象**：ALL 类型规则命中后直接 return，不再评估后续 DENY 规则，即使 stopProcessing=0
- **影响**：若用户期望"ALL 放行 + DENY 排除某些数据"，当前实现无法满足
- **建议**：明确 ALL 语义。若 ALL=完全放行则当前合理，UI 应提示；若 ALL 仅表示范围则需继续编排

### 问题 10：WorkflowAutoSkipService 用 Stream.toList() 导致运行时崩溃（已修复）
- **位置**：`WorkflowAutoSkipService.java:85`
- **现象**：`java.lang.Error: Unresolved compilation problem: The method toList() is undefined for the type Stream<Task>`
- **根因**：虽然 Java 17 支持 Stream.toList()，但编译环境不一致导致 class 文件含错误 stub
- **修复**：改为 `.collect(Collectors.toList())`
- **状态**：✅ 已修复

### 问题 11：实时跳过监听器与启动兜底重复执行（已修复）
- **位置**：`WorkflowAutoSkipService` 事件监听器 + `EntityDataService`/`EntityWorkflowRuntimeService` 启动兜底
- **现象**：启动兜底跳过节点后，ACTIVITY_STARTED 监听器再次尝试跳过同一任务，抛 "Task is already deleted" 错误
- **根因**：两处逻辑重复；且监听器在 ACTIVITY_STARTED 时任务可能尚未落库，查不到任务
- **修复**：保留启动兜底（保证第一个跳过节点被处理），监听器对 "already deleted" 异常降级为 DEBUG 日志
- **状态**：✅ 已修复，日志不再有 ERROR
