# 系统全面测试用例文档

## 一、测试概述

### 1.1 测试目标
验证工作流平台所有功能模块的数据正确性、前后端交互一致性以及数据库存储准确性。

### 1.2 测试范围
覆盖系统全部14个功能模块：
1. 登录认证（4个功能点）
2. 首页/工作台（6个功能点）
3. 流程管理（13个功能点）
4. 流程设计器（~40个用例，见独立文档）
5. 实体管理（18个功能点）
6. 表单设计（10个功能点）
7. 列表配置（7个功能点）
8. 数据管理（12个功能点）
9. 流程任务中心（20个功能点）
10. 系统管理（35个功能点）
11. 文件管理（4个功能点）
12. 扩展功能（8个功能点）

### 1.3 文档约定
- **TC** = Test Case（测试用例）
- **UI** = 用户界面
- **DB** = 数据库
- **API** = 后端接口
- **预期结果：PASS** = 操作成功且数据正确
- **预期结果：FAIL** = 操作应被拒绝并返回明确错误信息

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证使用正确用户名密码登录成功 |

```sql
  bpmn_xml LIKE '%bpmn:exclusiveGateway%' as has_gateway,

  bpmn_xml LIKE '%bpmn:sequenceFlow%' as has_flow
FROM process_definition_config
WHERE id='{processId}';
```
