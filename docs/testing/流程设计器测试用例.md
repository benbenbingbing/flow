# 流程设计器测试用例文档

## 一、测试概述

### 1.1 测试目标
验证流程设计器页面所有功能的数据保存准确性，确保前端设置正确持久化到数据库。

### 1.2 测试范围
- 所有节点类型（12种）
- 连接线（序列流）
- 右侧所有配置面板
- 数据库存储验证

### 1.3 数据库验证说明
所有测试用例都包含**预测数据库值**和**验证SQL**，执行测试后需要查询数据库验证。

---

### 2.1 开始事件（StartEvent）

| 项目 | 内容 |

## 四、集成测试用例

### 4.1 完整流程测试

#### TC-INTEGRATION-001 简单审批流程
| 项目 | 内容 |
|------|------|
| **测试目的** | 验证完整流程设计保存 |
| **测试场景** | 创建简单审批流程 |
| **操作步骤** | 1. 创建新流程<br>2. 拖拽开始事件（名称：开始）<br>3. 拖拽用户任务（名称：申请，执行人：申请人）<br>4. 拖拽用户任务（名称：审批，执行人：经理）<br>5. 拖拽结束事件（名称：结束）<br>6. 连接所有节点<br>7. 在审批节点绑定表单<br>8. 保存并发布流程 |
| **预测数据库验证** | 1. node_config：4条记录（START, USER_TASK*2, END）<br>2. assignee_config：2条记录（申请人和经理）<br>3. form_config：1条记录（审批表单）<br>4. process_definition_config：1条记录，status='PUBLISHED' |
| **验证SQL** | 见下方 |

**集成测试验证SQL：**
```sql
-- 验证节点配置
SELECT 
    nc.node_id, 
    nc.node_name, 
    nc.node_type,
    (SELECT COUNT(*) FROM assignee_config ac WHERE ac.node_config_id = nc.id) as assignee_count,
    (SELECT COUNT(*) FROM form_config fc WHERE fc.node_config_id = nc.id) as form_count
FROM node_config nc
WHERE nc.process_config_id = '流程ID'
ORDER BY nc.created_at;

-- 验证执行人配置
SELECT 
    nc.node_name,
    ac.assignee_type,
    ac.assignee_value
FROM assignee_config ac
JOIN node_config nc ON ac.node_config_id = nc.id
WHERE nc.process_config_id = '流程ID'
ORDER BY nc.node_name, ac.priority;

-- 验证表单配置
SELECT 
    nc.node_name,
    fc.form_name,
    fc.form_key,
    fc.is_readonly
FROM form_config fc
JOIN node_config nc ON fc.node_config_id = nc.id
WHERE nc.process_config_id = '流程ID';

-- 验证流程定义
SELECT process_key, version, status, bpmn_xml IS NOT NULL as has_xml
FROM process_definition_config
WHERE id = '流程ID';
```

## 六、自动化测试脚本

### 6.1 数据验证脚本

```sql
-- ========================================
-- 流程设计器数据验证脚本
-- 使用方法：执行流程保存后，替换'PROCESS_ID'为实际流程ID
-- ========================================

SET @process_id = 'PROCESS_ID';

-- 1. 验证节点数量
SELECT '节点数量' as check_item, 
       COUNT(*) as expected_value, 
       '实际节点数' as actual_value
FROM node_config 
WHERE process_config_id = @process_id;

-- 2. 验证节点类型分布
SELECT '节点类型分布' as check_item,
       node_type,
       COUNT(*) as count
FROM node_config 
WHERE process_config_id = @process_id
GROUP BY node_type;

-- 3. 验证执行人配置
SELECT '执行人配置' as check_item,
       nc.node_name,
       ac.assignee_type,
       ac.assignee_value,
       ac.priority
FROM assignee_config ac
JOIN node_config nc ON ac.node_config_id = nc.id
WHERE nc.process_config_id = @process_id
ORDER BY nc.node_name, ac.priority;

-- 4. 验证表单配置
SELECT '表单配置' as check_item,
       nc.node_name,
       fc.form_name,
       fc.form_key,
       fc.is_readonly
FROM form_config fc
JOIN node_config nc ON fc.node_config_id = nc.id
WHERE nc.process_config_id = @process_id;

-- 5. 验证节点跳过配置
SELECT '跳过配置' as check_item,
       node_name,
       skip_node
FROM node_config
WHERE process_config_id = @process_id AND skip_node = 1;

-- 6. 验证BPMN XML
SELECT 'BPMN XML' as check_item,
       LENGTH(bpmn_xml) as xml_length,
       CASE 
           WHEN bpmn_xml LIKE '%userTask%' THEN '包含userTask'
           ELSE '无userTask'
       END as has_user_task,
       CASE 
           WHEN bpmn_xml LIKE '%flowable:assignee%' THEN '包含assignee'
           ELSE '无assignee'
       END as has_assignee
FROM process_definition_config
WHERE id = @process_id;
```

## 七、常见问题排查

### 7.1 数据保存失败排查

| 现象 | 排查方法 | 可能原因 |
|------|----------|----------|
| 节点名称不保存 | 检查node_config表 | BPMN解析失败、ID重复 |
| 执行人不保存 | 检查assignee_config表、查看日志 | 正则不匹配（<userTask> vs <bpmn:userTask>） |
| 表单不保存 | 检查form_config表 | entityFormId解析失败、表单不存在 |
| 发布后数据丢失 | 对比发布前后的node_config | parseAndSaveNodeConfigs方法问题 |
| 多实例配置丢失 | 检查config_json字段 | JSON序列化问题 |
