# 功能疑问与建议

## 一、功能疑问

### 1. 种子数据文件
**疑问**: 测试期望 `V002__seed_builtin_data.sql` 种子数据文件，但项目中没有。V001 已是完整的业务 Schema 迁移文件。是否计划后续补充演示数据？

### 2. 审批面板交互
**位置**: `EntityApprovalDialog.vue`
**现状**: 审批意见面板展开后遮挡表单内容，当前无滚动机制
**待确认**: 应如何解决？（浮动覆盖 + 滚动？推挤内容区？弹出层？）

### 3. 数据库列命名 create_by vs created_by
**现状**: 系统统一使用 `create_by`（而非常见的 `created_by`）
**说明**: `create_by` 是整个系统一致使用的列名，测试中的 `created_by` 是错误期望

## 二、功能建议

### 1. 订阅方式增加搜索和选择
**建议**: 审批人配置的用户/组/角色选择器支持搜索过滤

### 2. 流程节点配置的校验
**建议**: 在保存/发布流程时校验所有审批节点是否已配置审批人

### 3. 前端单元测试覆盖
**当前**: 仅 calcEngine 有单元测试
**建议**: 增加 Vue 组件测试，特别是 NodeConfigPanel、审批对话框等核心组件

### 4. 测试报告自动生成
**建议**: 将后端/前端测试结果自动汇总为 HTML 报告

## 三、感觉应实现的功能

1. **批量操作** - 批量审批、批量转办
2. **流程版本回滚** - 发布后一键回滚到上一版本
3. **流程节点配置的冲突检测** - 未连接节点、循环引用检测
4. **计算引擎增强** - 增加日期函数、字符串处理、集合操作
5. **表单联动预览** - 流程节点配置表单时增加预览
